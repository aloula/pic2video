# GUI Contract: Multiplatform Desktop GUI

## Scope

Defines observable behavior for the desktop GUI that orchestrates the existing render workflow.

## Launch and Defaults

1. On app launch, input and output folder fields MUST default to the process working directory.
2. GUI MUST run on Windows, Linux, and macOS using the same functional behavior.

## Configuration Contract

1. GUI MUST allow selecting input and output folders.
2. GUI output selection MUST represent destination folder only; filename remains auto-generated by selected profile.
3. GUI MUST expose all current user-facing render options from the render workflow.

## Validation Contract

1. Before enabling/accepting Start, GUI MUST preflight validate required fields.
2. If input folder has no supported media, Start MUST be blocked with actionable message.
3. If output folder is invalid or unwritable, Start MUST be blocked with actionable message.
4. While a run is active, additional Start attempts MUST be blocked.

## Runtime Status Contract

1. Status indicator MUST show at least: `idle`, `loading files`, `processing`, `finished`, `failed`.
2. Status transitions MUST be visible to users in near real time.

## Log Box Contract

1. GUI MUST display a log box for runtime output lines.
2. New lines MUST append in chronological order.
3. On failure, error lines MUST be visible in the log box and status MUST become `failed`.

## Compatibility Contract

1. GUI-triggered runs MUST preserve current rendering semantics and option behavior.
2. Existing CLI workflow remains supported and unchanged.
